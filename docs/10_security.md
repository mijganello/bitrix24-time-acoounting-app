# Безопасность приложения

## Аутентификация и авторизация

### Как реализована защита

Приложение использует схему **JWT Bearer Token**:

1. Пользователь логинится → получает токен
2. Каждый защищённый запрос к API должен содержать этот токен
3. Сервер проверяет подпись токена

Зашита реализована через FastAPI Depends:
```python
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: Session = Depends(get_db),
) -> User:
```

Только эндпоинты `GET /api/auth/me` используют эту защиту. Остальные эндпоинты (отчёты, Bitrix24) — публичные, без авторизации. Это сделано намеренно: доступ к приложению контролируется организационно (внутренняя сеть или VPN).

## Хранение паролей

✅ Пароли хранятся в виде bcrypt-хеша с солью
✅ Исходный пароль никогда не сохраняется
✅ Параметр cost=12 делает брутфорс экономически нецелесообразным
✅ При сравнении используется `bcrypt.checkpw` — защищён от timing attack

---

## JWT-токены

✅ Токен подписан с использованием `SECRET_KEY` (HS256)
✅ Токен содержит время истечения (`exp`)
✅ Стандартный срок жизни: 8 часов
✅ При logout токен удаляется из localStorage клиента

⚠️ **Ограничение**: токены нельзя отозвать без сохранения blacklist на сервере. После logout старый токен технически действителен до истечения срока. Для учебного/внутреннего приложения это приемлемо.

## CORS (Cross-Origin Resource Sharing)

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=[os.getenv("FRONTEND_URL", "*")],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

В разработке используется `*` (все источники). В продакшн `FRONTEND_URL` задаётся конкретным доменом фронтенда.

## OWASP Top 10 — проверка

| Уязвимость | Статус | Детали |
|---|---|---|
| A01: Broken Access Control | ✅ | Защищённые маршруты требуют валидного JWT |
| A02: Cryptographic Failures | ✅ | bcrypt для паролей, HS256 для JWT |
| A03: Injection | ✅ | SQLAlchemy ORM предотвращает SQL-инъекции |
| A04: Insecure Design | ✅ | Секреты в env, пароли хешируются |
| A05: Security Misconfiguration | ⚠️ | CORS `*` в dev — нормально. В prod задавать конкретный origin |
| A07: Identification Failures | ✅ | Блокировка по `is_active`, проверка токена при каждом запросе |
| A09: Security Logging | ⚠️ | Базовое логирование Uvicorn. Отдельный аудит-лог не реализован |
