# Используемые технологии — полный список

## Языки программирования

### Python 3.12
Язык программирования общего назначения. Используется для написания серверной части (backend). Выбран за:
- Богатую экосистему библиотек для веб-разработки
- Читаемый синтаксис
- Отличную поддержку асинхронного программирования (async/await)
- Широкое применение в индустрии

### JavaScript (ECMAScript 2022+)
Язык программирования для браузера. Используется для написания клиентской части (frontend). В проекте используется современный синтаксис: ES-модули, стрелочные функции, deconstructuring, optional chaining.

### JSX (JavaScript XML)
Расширение синтаксиса JavaScript, разработанное командой React. Позволяет писать HTML-подобную разметку прямо в JavaScript-коде. Компилируется в обычный JavaScript сборщиком Vite.

## Backend — библиотеки Python

### FastAPI (0.115+)
Современный веб-фреймворк для создания REST API. Ключевые особенности:
- **Автоматическая документация**: Swagger UI по `/docs`, ReDoc по `/redoc`
- **Типизация**: использует Python type hints для валидации запросов/ответов
- **Асинхронность**: полная поддержка `async/await`
- **Dependency Injection**: удобная система внедрения зависимостей (`Depends`)
- **Скорость**: один из самых быстрых Python-фреймворков

Пример эндпоинта:
```python
@router.get("/users-summary")
async def report_users_summary(
    date_from: date = Query(...),   # Обязательный параметр
    date_to: date = Query(...),
    user_id: Optional[int] = Query(None),  # Необязательный
):
    ...
```

FastAPI автоматически:
- Парсит `date_from` из строки `"2024-03-01"` в объект `date`
- Возвращает 422, если параметр не передан или неверного типа
- Генерирует схему OpenAPI для Swagger UI

### Uvicorn (0.34+)
ASGI-сервер (Asynchronous Server Gateway Interface). Это программа, которая принимает TCP-соединения и передаёт HTTP-запросы в FastAPI. Аналог Gunicorn, но с поддержкой асинхронного кода.

В разработке запускается с `--reload` (перезапуск при изменении py-файлов). В продакшн — без reload, возможно несколько worker-процессов.

### SQLAlchemy 2.0+
ORM для работы с реляционными базами данных. Версия 2.0 — новый стиль API (не legacy). Используется в «Core+ORM»-режиме: модели как классы Python, запросы через сессии.

Позволяет сменить СУБД без изменения кода — достаточно изменить строку подключения.

### Pydantic 2.0+
Библиотека валидации данных с использованием Python type hints. FastAPI использует Pydantic для:
- Проверки входящих данных запроса
- Сериализации ответов в JSON
- Схем для Swagger UI

```python
class UserOut(BaseModel):
    id: int
    username: str
    is_active: bool
    avatar_color: str | None = None

    model_config = {"from_attributes": True}
    # from_attributes позволяет создать из SQLAlchemy-объекта
```

### python-jose
Реализация стандартов JWT (JSON Web Tokens) и JWS/JWE. Используется для:
- Создания JWT-токенов при логине
- Верификации токена при каждом запросе
- Алгоритм подписи: HS256 (HMAC с SHA-256)

### bcrypt
Библиотека для криптографического хеширования паролей. Алгоритм bcrypt специально разработан для хеширования паролей:
- Медленный (намеренно) — защита от брутфорса
- Со «стоимостью» (cost factor) — можно увеличивать с ростом мощностей железа
- С встроенной солью — каждый хеш уникален

### httpx
Современный HTTP-клиент для Python с поддержкой async/await. Аналог requests, но асинхронный. Используется для запросов к Bitrix24 REST API.

```python
async with httpx.AsyncClient(timeout=30.0) as client:
    resp = await client.post(f"{webhook}/tasks.task.list.json", json={...})
```

### psycopg2-binary
Драйвер для подключения Python к PostgreSQL. «Binary» = предкомпилированная версия без необходимости компилировать C-код. Используется SQLAlchemy «под капотом».

### python-dotenv
Загружает переменные окружения из файла `.env`. В Docker-окружении не обязательна (переменные передаются через `environment:` в compose), но удобна для локального запуска без Docker.

## Frontend — библиотеки JavaScript

### React 19
Библиотека для построения пользовательских интерфейсов. Разработана Facebook (Meta). Ключевые концепции:
- **Компоненты**: независимые блоки UI (функции, возвращающие JSX)
- **State**: локальное состояние компонента (`useState`)
- **Props**: параметры, передаваемые компоненту от родителя
- **Хуки (Hooks)**: функции для использования возможностей React в функциональных компонентах
- **Virtual DOM**: React хранит виртуальное представление DOM и обновляет только изменившееся

### React Router DOM 7
Маршрутизация в браузере (SPA — Single Page Application). Позволяет создавать навигацию без полной перезагрузки страницы. Компоненты:
- `BrowserRouter` — провайдер, использует HTML History API
- `Routes / Route` — описание маршрутов
- `Link / Navigate` — навигация
- `useNavigate` — программный переход
- `useLocation` — текущий URL

### Ant Design (antd) 6.x
Компонентная библиотека с богатым набором готовых UI-компонентов для React. Разработана командой Alibaba. В проекте используются:

| Компонент | Применение |
|---|---|
| `Table` | Таблицы с сортировкой, раскрывающимися строками |
| `DatePicker / RangePicker` | Выбор даты и диапазона дат |
| `Select` | Выпадающий список |
| `Button` | Кнопки |
| `Card` | Карточки-панели |
| `Statistic` | Большие числа с подписью |
| `Progress` | Прогресс-бар |
| `Breadcrumb` | Хлебные крошки (навигация) |
| `Alert` | Информационные плашки |
| `Tag` | Метки |
| `Tooltip` | Всплывающие подсказки |
| `Space` | Горизонтальные/вертикальные отступы |
| `Form` | Форма с валидацией |
| `Menu` | Навигационное меню |
| `Spin` | Индикатор загрузки |
| `ConfigProvider` | Глобальная тема |
| `Empty` | Заглушка при отсутствии данных |

### dayjs
Библиотека для работы с датами. Облегчённая альтернатива moment.js (2 KB vs 67 KB). Используется для:
- Форматирования дат
- Вычисления начала/конца недели/месяца (пресеты в DatePicker)
- Подсчёта рабочих дней
- Определения времени суток для приветствия

### Vite 6
Инструмент сборки нового поколения. В отличие от webpack:
- В dev-режиме: сервирует исходники как ES-модули напрямую, без бандлинга → мгновенный старт
- HMR: обновляет только изменившиеся модули, не весь бандл
- В prod: использует Rollup для оптимизированной сборки

Плагин `@vitejs/plugin-react` добавляет:
- Обработку JSX (компиляция через Babel)
- React Fast Refresh (аналог HMR для React-компонентов)

## Инфраструктура

### Docker
Платформа контейнеризации. Изолирует каждый сервис в отдельный контейнер с собственной файловой системой, сетевым стеком и набором зависимостей.

### Docker Compose
Инструмент управления несколькими Docker-контейнерами. Описывает всю инфраструктуру в файле `docker-compose.yml`.

### PostgreSQL 16
Мощная реляционная СУБД с открытым исходным кодом. Поддерживает ACID-транзакции, полнотекстовый поиск, JSON-поля, геоданные и многое другое. В проекте используется через SQLAlchemy.

### Nginx 1.27
Высокопроизводительный веб-сервер, обратный прокси. В проекте:
- Development: обратный прокси для backend и frontend
- Production: раздача статических файлов frontend + проксирование API

## Протоколы и стандарты

| Стандарт | Применение |
|---|---|
| **HTTP/1.1** | Основной протокол взаимодействия клиент-сервер |
| **WebSocket** | HMR Vite в режиме разработки |
| **OAuth2 (Password Flow)** | Схема авторизации (логин/пароль → токен) |
| **JWT (RFC 7519)** | Формат токенов авторизации |
| **HS256** | Алгоритм подписи JWT (HMAC + SHA-256) |
| **OpenAPI 3.x** | Спецификация API, генерируется FastAPI автоматически |
| **CORS** | Политика безопасности браузера для кросс-доменных запросов |
| **REST** | Архитектурный стиль API |
| **JSON** | Формат данных между клиентом, сервером и Bitrix24 |
| **ISO 8601** | Формат дат и времени (`2024-03-15T14:30:00+03:00`) |
