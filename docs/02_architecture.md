# Архитектура приложения

## Общая схема

Приложение состоит из **четырёх сервисов**, каждый из которых запускается в отдельном Docker-контейнере. Docker — это технология, позволяющая упаковать программу со всеми её зависимостями в изолированный «контейнер» и запустить на любом сервере.

```
Браузер пользователя
        │
        ▼
    ┌─────────┐
    │  Nginx  │ ← единственная «дверь» снаружи, порт 80
    └────┬────┘
         │
    ┌────┴──────────────────────┐
    │                           │
    ▼                           ▼
┌──────────────┐       ┌──────────────────┐
│   Frontend   │       │     Backend      │
│ (React + Vite│       │ (FastAPI/Python) │
│  порт 5173)  │       │   порт 8000)     │
└──────────────┘       └────────┬─────────┘
                                │
                                ▼
                       ┌──────────────────┐         ┌──────────────────┐
                       │   PostgreSQL DB  │         │    Bitrix24      │
                       │   порт 5432      │         │ (внешний портал) │
                       └──────────────────┘         └──────────────────┘
```

### Как работает схема

1. Пользователь открывает браузер и набирает адрес сервера.
2. **Nginx** принимает запрос. Nginx — это высокопроизводительный веб-сервер, который выступает в роли «диспетчера»:
   - Если запрос начинается с `/api/` → пересылает его на **Backend** (FastAPI, порт 8000).
   - Все остальные запросы → пересылает на **Frontend** (Vite dev-сервер, порт 5173).
3. **Frontend** — React-приложение. Отдаёт пользователю HTML, CSS и JavaScript-код интерфейса. После загрузки в браузере работает самостоятельно.
4. **Backend** — Python-приложение на FastAPI. Принимает запросы от фронтенда, обращается к БД и к Bitrix24, возвращает данные в формате JSON.
5. **PostgreSQL** — база данных. Хранит только пользователей системы (логины, хэши паролей). Данные о задачах, проектах и времени берутся напрямую из Bitrix24 при каждом запросе, не кэшируются.
6. **Bitrix24** — внешний сервис. Backend обращается к нему через **Webhook** (специальный URL с ключом доступа). Bitrix24 возвращает данные: задачи, затраченное время, пользователей, проекты.

## Docker Compose — оркестрация сервисов

Файл `docker-compose.yml` описывает все четыре сервиса и их связи.

```yaml
services:
  db:         # PostgreSQL база данных
  backend:    # Python FastAPI
  frontend:   # React + Vite
  nginx:      # Nginx (обратный прокси)
```

Каждый сервис описывается параметрами:
- **image** — готовый образ из Docker Hub (для PostgreSQL и Nginx).
- **build** — папка с `Dockerfile` (для backend и frontend) — инструкция как собрать образ.
- **environment** — переменные окружения (настройки: URL базы данных, пароли и др.).
- **depends_on** — зависимости: nginx запускается только после backend и frontend.
- **volumes** — смонтированные папки: изменения в коде сразу отражаются в контейнере (для режима разработки).

## Взаимодействие сервисов (подробно)

### Nginx как обратный прокси

Nginx использует конфигурацию `nginx/nginx.conf`. В ней три ключевых блока:

```nginx
# Запросы /api/* → Backend
location /api/ {
    proxy_pass http://backend:8000;
    ...
}

# WebSocket для HMR (горячая перезагрузка кода в режиме разработки)
location /vite-hmr {
    proxy_pass http://frontend:5173;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    ...
}

# Всё остальное → Frontend
location / {
    proxy_pass http://frontend:5173;
    ...
}
```

Благодаря такой конфигурации, пользователь видит один адрес (`http://...:80`), не зная о существовании отдельных сервисов.

Имена `backend` и `frontend` — это имена Docker-сервисов. Docker автоматически резолвит их в IP-адреса контейнеров (встроенный DNS).

### Swagger UI / OpenAPI

Nginx также проксирует `/docs` и `/openapi.json` — это автоматически генерируемая FastAPI документация API. По адресу `http://localhost/docs` можно увидеть и протестировать все API-методы в браузере прямо без фронтенда.

## Переменные окружения (.env)

Секретные настройки хранятся в файле `.env` (не коммитится в Git). Пример:

```
BITRIX24_WEBHOOK_URL=https://mycompany.bitrix24.ru/rest/1/xxxxxxxx/
SECRET_KEY=super-secret-key-change-in-production
ACCESS_TOKEN_EXPIRE_MINUTES=480
FRONTEND_URL=http://localhost
DATABASE_URL=postgresql://bitrix_user:bitrix_pass@db:5432/bitrix_app
```

Backend читает эти переменные при старте. `python-dotenv` — библиотека, которая загружает переменные из `.env` файла.
