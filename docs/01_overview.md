# Общее описание приложения WORKERPUNCH v2

## Что это такое?

**WORKERPUNCH v2** — это веб-приложение для учёта рабочего времени сотрудников, которые пользуются корпоративным порталом **Bitrix24**. Bitrix24 — это популярная в России платформа для управления бизнесом: там ставятся задачи, ведутся проекты, хранятся контакты. У каждой задачи в Bitrix24 сотрудники могут фиксировать потраченное время (called «затраченное время» или «elapsed time»).

Проблема: в самом Bitrix24 аналитика по затраченному времени либо неудобная, либо доступна только в платных тарифах. WORKERPUNCH решает эту проблему — он забирает данные из Bitrix24 через специальный API (интерфейс для программного взаимодействия), агрегирует их и показывает красивые отчёты: кто сколько отработал, по каким задачам, по каким проектам, в какие дни.

## Название

**WORKERPUNCH** — от английских слов «worker» (работник) и «punch» (штамповать, отмечаться). Это отсылка к старым карточным часам, куда рабочие «пробивали» свою карточку при приходе и уходе.

Текущая версия: **0.5.1-alpha** (ранняя тестовая версия, в разработке).

## Как выглядит приложение для конечного пользователя?

Пользователь открывает в браузере адрес сервера (например, `http://localhost` или корпоративный домен). Перед ним появляется страница логина с эффектом «стекла» (glassmorphism). После ввода логина и пароля пользователь попадает в главный интерфейс с тремя зонами:

1. **Боковая панель (Sidebar)** слева — меню навигации: «Главная», «Отчёты», «Статус платформы». Панель можно свернуть для экономии места.
2. **Шапка (Header)** сверху — логотип WORKERPUNCH и имя текущего пользователя с кнопкой выхода.
3. **Центральная контентная область** — здесь отображаются страницы с отчётами.

Дизайн выполнен в стиле glassmorphism: полупрозрачные белые панели с эффектом матового стекла (backdrop-filter: blur), мягкие тени, скруглённые углы.

## Проблема, которую решает приложение

| Задача | Без WORKERPUNCH | С WORKERPUNCH |
|---|---|---|
| Узнать, сколько часов отработал сотрудник за месяц | Нужно заходить в каждую задачу в Bitrix24 | Один клик — таблица с часами по всем сотрудникам |
| Проверить нагрузку команды по дням | Нет встроенного инструмента | Тепловая карта: цветом виден каждый день каждого сотрудника |
| Увидеть, сколько времени ушло на конкретный проект | Тяжело агрегировать вручную | Отчёт «По проектам»: дерево проект → задача → исполнитель |
| Посмотреть личную статистику | Только через интерфейс Bitrix24 | Личный дашборд с топом задач и графиком по дням |

## Версии и окружения

Приложение поддерживает два режима запуска:

- **Development (разработка)** — запускается командой `docker-compose up`. Используется для разработки и тестирования. Фронтенд работает с горячей перезагрузкой кода (HMR — Hot Module Replacement).
- **Production (боевой)** — запускается через `docker-compose.prod.yml`. Фронтенд собирается в статические файлы и раздаётся через Nginx напрямую.
